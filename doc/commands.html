The File Menu

File → New (Ctrl+N)

Creates a new setlist

File → Open (Ctrl+O)

Opens a setlist

File → Save (Ctrl+S)

Saves the setlist

File → Save As (Ctrl+Shift+S)

Saves the setlist to a specified location

The Settings and Help Menu

Performer has the common KDE Settings and Help menu items, for more information read the sections about the Settings Menu and Help Menu of the KDE Fundamentals.

Apart from the common KDE menus described in the Menu chapter of the KDE Fundamentals documentation Performer has these application specific menu entries:

Settings → MIDI Program Changes

If enabled, the setlist will react to MIDI Program Change messages. This means that if a MIDI Program Change message is received at the control_gui-in port, the Play Now-action of the corresponding song in the setlist will be triggered.

Settings → Always on top

The main window of Performer always stays on top if this is enabled.

Settings → Hide Carla

Each song in a setlist of Performer can have a Carla patch assigned. If a song is active, an instance of Carla will be started, loading the corresponding patch of the song. If this menu item is enabled, the instances of Carla started by Performer are invisible. If it is disabled, the instances of Carla are shown as regular application windows.

Settings → Show MIDI controls

Show a dock widget with settings for MIDI controls. MIDI CC messages received at the control_gui-in port can control actions of Performer. The MIDI control dock widget can be used to map MIDI CC messages to actions of Performer
